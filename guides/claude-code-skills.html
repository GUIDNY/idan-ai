Skills ב-Claude Code: איך להפוך אותו למומחה בפרויקט שלכם

מאת עידן גינדי עודכן יולי 2026 זמן קריאה: 5 דקות

Claude Code הוא Claude שרץ בטרמינל ועובד ישירות על בסיסי קוד. אבל לכל צוות יש דרך עבודה משלו - איך מריצים בדיקות, איך בונים פיצ'ר, אילו מוסכמות קוד נהוגות. Skills הוא המנגנון שמלמד את Claude Code את המוסכמות הספציפיות שלכם.

מה זה בפועל

Skill הוא תיקייה עם קובץ הוראות (SKILL.md) ש-Claude Code טוען לפי הצורך - כשהוא מזהה שהמשימה רלוונטית לאותו Skill. במקום להסביר בכל שיחה מחדש "אצלנו בונים פיצ'ר ב-Next.js ככה", ה-Skill כבר "יודע" את זה.

איך מתקינים: יוצרים תיקייה תחת .claude/skills/ בפרויקט (או ~/.claude/skills/ גלובלית), כותבים בה קובץ SKILL.md עם ההוראות, ומפעילים מחדש את Claude Code. תוך דקות אחדות ה-Skill פעיל.

Skills מול Hooks מול Subagents

שלושת המנגנונים משלימים זה את זה, לא מתחרים:

  • Skills - נותן ידע ומומחיות חוזרת ("ככה עושים דבר X אצלנו").
  • Hooks - נותן תגובות דטרמיניסטיות לאירועים ("תריץ פורמטר אחרי כל שמירה") - קוד קבוע, לא תלוי בפרשנות המודל.
  • Subagents - נותן חלונות הקשר מבודדים ("תעביר את תת-המשימה הזו לסוכן נפרד").

למה זה משתלם

בלי Skill, כל שיחה חדשה עם Claude Code מתחילה בלי ידע על ההקשר הארגוני שלכם. עם Skill מוגדר היטב, כל מפתח בצוות שמפעיל את Claude Code על הפרויקט מקבל את אותה "מומחיות" - עקביות גבוהה יותר, פחות הסברים חוזרים.

Claude Projects: איך לארגן שיחות וידע במקום אחד

מאת עידן גינדי עודכן יולי 2026 זמן קריאה: 7 דקות

מי שעובד עם Claude על יותר מנושא אחד מכיר את הבעיה: עשרות שיחות בלי סדר, וכשרוצים לחזור למידע ממקום מסוים - קשה למצוא אותו. Projects נועד בדיוק לזה.

מה זה בפועל

Project הוא סוג של "תיקייה חכמה" בתוך Claude.ai: מרחב עבודה קבוע שאפשר להעלות אליו קבצים, מסמכים וקוד - וכל שיחה בתוך אותו Project "רואה" את כל החומר הזה כבסיס ידע משותף, בלי צורך להעלות אותו מחדש בכל פעם.

דוגמה מעשית: אם אתם מלווים כמה לקוחות, אפשר לפתוח Project נפרד לכל לקוח - עם המסמכים, הטון והקונטקסט הספציפיים שלו. כשפותחים שיחה חדשה בתוך אותו Project, Claude כבר "מכיר" את הרקע, בלי לערבב מידע בין לקוחות שונים.

Projects מול זיכרון רגיל

ההבדל המרכזי: הזיכרון הוא גלובלי ועובר איתכם בין כל השיחות, בעוד ש-Project הוא הקשר מבודד ומכוון - מתאים במיוחד כשרוצים להפריד בין תחומים שונים (למשל עבודה מול פרויקט אישי) או לשמור בסיס ידע מסודר לצוות שלם.

מתי כדאי להשתמש בזה

  • עבודה מתמשכת על אותו קוד/מוצר לאורך זמן
  • ליווי כמה לקוחות/פרויקטים במקביל
  • בסיס ידע משותף לצוות - כולם עובדים מול אותו Project ואותו מידע

איך מקימים Project ראשון - צעד אחר צעד

  1. יוצרים Project חדש ונותנים לו שם ברור (שם הלקוח, שם המוצר).
  2. מעלים אליו את המסמכים הרלוונטיים - בריף, מסמכי מדיניות, קוד קיים, דוגמאות סגנון.
  3. אפשר להוסיף גם הוראות מותאמות אישית ברמת ה-Project עצמו - כך שכל שיחה בתוכו "יודעת" גם את הטון והכללים הספציפיים, לא רק את המסמכים.
  4. פותחים שיחה חדשה בתוך ה-Project בכל פעם שעובדים על הנושא הזה, במקום שיחה כללית.

Projects לעבודה אישית מול עבודת צוות

אצל יחיד, Project הוא בעיקר כלי ארגון - תיקייה חכמה לכל נושא. בעבודת צוות, זה הופך לבסיס ידע משותף: כל חבר צוות שנכנס לאותו Project מקבל את אותו הקשר בדיוק, במקום שכל אחד "יסביר" לכלי מחדש. זה שימושי במיוחד כשיש תחלופת אנשי צוות - הידע נשאר ב-Project, לא רק אצל מי שהתחיל אותו.

שאלות נפוצות

כמה Projects אפשר לפתוח?

המספר המדויק תלוי בתוכנית שבה אתם משתמשים - שווה לבדוק את המגבלות העדכניות בהגדרות החשבון שלכם.

מה קורה אם מעלים יותר מדי מסמכים ל-Project אחד?

ריבוי מסמכים לא רלוונטיים יכול "לדלל" את ההקשר ולפגוע בדיוק התשובות. עדיף Project ממוקד עם המסמכים הרלוונטיים באמת, מאשר "תיקיית כל-בו" עם הכל.

האם אפשר לשתף Project עם אנשים אחרים?

בתוכניות צוות/ארגון כן קיימת אפשרות שיתוף - כדאי לבדוק את ההגדרות הספציפיות לתוכנית שלכם כדי לדעת בדיוק איך זה עובד ומי רואה מה.

רוצים לבנות מבנה Projects מסודר לעבודה שלכם או של הצוות? בואו נדבר.
